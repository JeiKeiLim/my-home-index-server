Home Port Manager — System Architecture

Single Go binary running on the user's MacBook. Web UI served at yourhost.example:40000.

inspect / kill / spawn (magenta paths) HTTP / data flow user-owned TCP listener external process (no log capture) self (excluded from UI actions)
Critical path: every 2s, the Port Scanner walks own-uid PIDs via libproc (no subprocess, ~10ms), collects all TCP listeners in 40000–40500, enriches them through Process Inspector, and the HTTP router renders the table. The dashboard excludes its own PID.
Actions: kill sends SIGTERM → waits 3s → SIGKILL fallback; replay re-spawns via os/exec with detached pgid, piping stdout+stderr into the ring buffer for that PID; SSE streams to the browser.
External processes (launched outside the dashboard) show all metadata but their log rows display "logs unavailable — relaunch via dashboard to capture" per the macOS log-capture feasibility research.
